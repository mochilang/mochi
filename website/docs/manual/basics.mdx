This page provides a quick overview of the Mochi language — its key syntax and concepts.

If you know Python, a lot of Mochi code will look familiar. Mochi adds static type checking, value safety, and unique constructs like agents, streams, and native AI generation.

## Hello, world

```mochi
print("Hello, world!")
```

No `main()` function needed. Mochi programs execute top to bottom.

## Variables

Use `let` for immutable bindings (the default) and `var` for mutable ones:

```mochi
let name = "Mochi"    // immutable — cannot be reassigned
var count = 0         // mutable

count = count + 1
```

The type is inferred from the value. You can also annotate explicitly:

```mochi
let x: int = 42
let ratio: float = 3.14
```

For more, see [Variables](/docs/manual/variables).

## Functions

Functions are declared with `fun`:

```mochi
fun add(a: int, b: int): int {
  return a + b
}
```

Arrow syntax for single-expression functions:

```mochi
let square = fun(x: int): int => x * x
```

Functions are first-class values:

```mochi
fun apply(f: fun(int): int, n: int): int {
  return f(n)
}

print(apply(square, 5))   // 25
```

For more, see [Functions](/docs/manual/functions).

## Types

Mochi's primitive types: `int`, `float`, `string`, `bool`.

Define custom types with `type`:

```mochi
type Point {
  x: float
  y: float
}

let p = Point { x: 1.0, y: 2.0 }
print(p.x)   // 1.0
```

Union types for nullable or multi-type values:

```mochi
let maybe: string | nil = nil
```

For more, see [Types](/docs/manual/types).

## Collections

Lists, maps, and sets are built in:

```mochi
let nums = [1, 2, 3, 4, 5]
let user = {"name": "Ana", "age": 30}
let tags = {"go", "python", "mochi"}

print(nums[0])           // 1
print(user["name"])      // Ana
print("go" in tags)      // true
```

Destructuring:

```mochi
let [a, b] = [1, 2]
let {"name": n} = {"name": "Ana"}
```

## Control flow

```mochi
if x > 0 {
  print("positive")
} else if x < 0 {
  print("negative")
} else {
  print("zero")
}

for i in 0..5 {
  print(i)
}

var n = 10
while n > 0 {
  n = n - 1
}

match x {
  0 => print("zero"),
  1 => print("one"),
  _ => print("other"),
}
```

`if` can also be used as an expression:

```mochi
let label = if x > 0 then "pos" else "non-pos"
```

For more, see [Control flow](/docs/manual/control-flow).

## Packages

Organize code into packages with `package`:

```mochi
package math

export fun abs(n: int): int {
  return if n < 0 then -n else n
}
```

Import with `import`:

```mochi
import math

print(math.abs(-5))   // 5
```

For more, see [Packages](/docs/manual/packages).

## Tests

Built-in testing with `test` and `expect`:

```mochi
test "basic arithmetic" {
  expect 1 + 1 == 2
  expect 10 / 2 == 5
}
```

Run with:

```bash
mochi test myfile.mochi
```

## Agents

React to events with `agent`:

```mochi
agent logger {
  on message(text: string) {
    print("[log]", text)
    emit logged(text)
  }
}
```

For more, see [Agents](/docs/manual/agents).

## AI generation

Use `generate` to call AI models:

```mochi
let answer = generate text {
  prompt: "What is 2 + 2?"
}
print(answer)
```

For more, see [Generative AI](/docs/manual/generative-ai).
