This page tracks the direction of the Mochi language. It is directional guidance, not a fixed release schedule. Priorities shift based on community feedback.

:::note Status indicators

- Done
- In progress
- Planned

:::

## v0.x (current development)

Active milestones.

### Language core

- Done. Bytecode VM with constant folding.
- Done. Static type inference.
- Done. `let` / `var` bindings.
- Done. Functions, closures, arrow syntax.
- Done. `type` structs.
- Done. Union types (`T | nil`, `A | B`).
- Done. `for`, `while`, `if/else`, `match`.
- Done. Lists, maps, sets.
- Done. Packages and imports.
- Done. Built-in test blocks (`test` / `expect`).
- Done. Agents and streams.
- Done. `generate` blocks for AI generation.
- Done. Dataset queries (`from` / `where` / `select`).
- Done. FFI through `extern` declarations.
- In progress. Generics.
- Planned. Error propagation operator (`?`).
- Planned. Async / await primitives.
- Planned. Interfaces and protocols.

### Tooling

- Done. `mochi run` executes programs.
- Done. `mochi test` runs test blocks.
- Done. `mochi build` compiles to a binary.
- Done. `mochi serve` runs MCP server mode.
- Done. Docker image (`ghcr.io/mochilang/mochi`).
- Done. VS Code syntax highlighting (`mochi-tm`).
- In progress. Language server (LSP).
- Planned. Formatter (`mochi fmt`).
- Planned. Package manager and registry.
- Planned. REPL improvements.
- Planned. Debugger.

### Transpilation

- Done. Transpile to Go.
- Done. Transpile to Python.
- Done. Transpile to TypeScript.
- Planned. Transpile to Lua.

### Platform

- Done. macOS (Intel and Apple Silicon).
- Done. Linux (x86_64, ARM64).
- Done. Docker.
- In progress. Native Windows (without WSL).
- Planned. WebAssembly.

## v1.0 (stable release)

The 1.0 milestone marks language stability. No breaking changes to the core language after this point.

To reach 1.0, Mochi needs:

- Language specification v1.0.
- Generics finalized.
- Stable standard library API.
- LSP with IDE integration for VS Code and JetBrains.
- Package registry.
- Native Windows support.
- Comprehensive test suite.

## Community priorities

User feedback shapes the priorities above. File requests in [GitHub Discussions](https://github.com/mochilang/mochi/discussions).
