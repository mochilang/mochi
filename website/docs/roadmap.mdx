This page provides a high-level roadmap for how Mochi will evolve. It is **directional guidance**, not a fixed release schedule. Priorities shift as we learn from the community.

:::note Status indicators

- ✅ Done
- 🚧 In progress
- ⬜ Planned

:::

## v0.x — Current development

These are the milestones we're actively working through.

### Language core

- ✅ Bytecode VM with constant folding
- ✅ Static type inference
- ✅ `let` / `var` bindings
- ✅ Functions, closures, arrow syntax
- ✅ `type` structs
- ✅ Union types (`T | nil`, `A | B`)
- ✅ `for`, `while`, `if/else`, `match`
- ✅ Lists, maps, sets
- ✅ Packages and imports
- ✅ Built-in test blocks (`test` / `expect`)
- ✅ Agents and streams
- ✅ `generate` blocks (AI generation)
- ✅ Dataset queries (`from`/`where`/`select`)
- ✅ FFI (`extern` declarations)
- 🚧 Generics
- ⬜ Error propagation operator (`?`)
- ⬜ Async/await primitives
- ⬜ Interfaces / protocols

### Tooling

- ✅ `mochi run` — execute programs
- ✅ `mochi test` — run test blocks
- ✅ `mochi build` — compile to binary
- ✅ `mochi serve` — MCP server mode
- ✅ Docker image (`ghcr.io/mochilang/mochi`)
- ✅ VS Code syntax highlighting (`mochi-tm`)
- 🚧 Language server (LSP)
- ⬜ Formatter (`mochi fmt`)
- ⬜ Package manager / registry
- ⬜ REPL improvements
- ⬜ Debugger

### Transpilation

- ✅ Transpile to Go
- ✅ Transpile to Python
- ✅ Transpile to TypeScript
- ⬜ Transpile to Lua

### Platform

- ✅ macOS (Intel + Apple Silicon)
- ✅ Linux (x86_64, ARM64)
- ✅ Docker
- 🚧 Windows (native, without WSL)
- ⬜ WebAssembly

## v1.0 — Stable release

The 1.0 milestone represents language stability: no breaking changes to the core language after this point.

To reach 1.0, we need:

- ⬜ Language specification v1.0
- ⬜ Generics finalized
- ⬜ Stable standard library API
- ⬜ LSP with IDE integration for VSCode and JetBrains
- ⬜ Package registry
- ⬜ Native Windows support
- ⬜ Comprehensive test suite

## Community priorities

The priorities above are influenced by what we hear from users. Tell us what matters most to you in [GitHub Discussions](https://github.com/mochilang/mochi/discussions).
