Use the in-place index-placement idea. For each position, while the current value `v` is in the range `1..n` and is not already sitting at index `v-1`, swap it into that target slot. After this cleanup pass, the first index `i` where `nums[i] != i + 1` gives the smallest missing positive `i + 1`. If every slot is correct, the answer is `n + 1`. This stays `O(n)` because each swap places at least one value into its final slot, and it uses `O(1)` extra space because everything happens inside the input array.
