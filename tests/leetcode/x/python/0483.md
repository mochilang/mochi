If `n = 1 + k + k^2 + ... + k^m`, then the smallest good base is the smallest `k`, which means the largest possible digit count `m + 1`. Iterate the possible lengths from large to small, binary-search the base for each length, and evaluate the geometric sum with overflow guards; the first exact match is the answer. If no longer representation exists, the fallback is `n - 1`, which always gives `11` in base `n - 1`.
