Let `dp[j]` be the number of arrays using `1..i-1` with exactly `j` inverse pairs. When inserting `i`, it can create anywhere from `0` to `i-1` new inverse pairs depending on where it is placed, so `next[j] = sum(dp[j-x])` over that range. A sliding window turns that sum into O(1) per state: as `j` increases, add the new right endpoint and subtract the value that falls out of the window. This yields O(nk) time and O(k) space.
