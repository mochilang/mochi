Work backward with dynamic programming. Let `dp[i]` be the minimum total cost to reach the end from index `i`, and let `path[i]` store the chosen 1-based path from `i` onward. For each reachable position, try every next jump within `maxJump`, keep the smaller total cost, and when costs tie keep the lexicographically smaller full path. Building from the end toward the front makes each decision depend only on already solved suffixes.
