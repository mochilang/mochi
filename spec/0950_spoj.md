# SPOJ Problems in Mochi

Goal: solve classical SPOJ problems in Mochi to demonstrate the language's expressive power and validate the runtime.

## Directory layout

```
tests/spoj/x/mochi/
  00000/    # problems 1-99
    1.md    # problem description / approach
    1.mochi # Mochi solution
    1.in    # sample input
    1.out   # expected output
    ...
  00100/    # problems 100-199
  00200/    # problems 200-299
  ...
```

Each subdir groups 100 problem IDs. Files are named by the SPOJ numeric ID (not the slug).

## Implementation rules

- Each `.mochi` file must compile and run correctly against the sample `.in`.
- Read from `stdin` using `input()`. Print to stdout using `print()`.
- No external libraries - only Mochi builtins.
- Keep solutions simple and readable; avoid micro-optimizations unless needed for correctness.
- Each `.md` file briefly states: problem summary, algorithm chosen, edge cases.
